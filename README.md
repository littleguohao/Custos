# strategy_team

持续进化的投资策略 Team。

## 核心目标

辅助完成：

- 市场择时
- 产业研究
- 主线/板块判断
- 选股池
- 买入计划
- 持仓研判
- 卖出风控
- 总控决策
- 交易复盘与策略进化

## 生产架构

```mermaid
flowchart TD
    A[确定性数据采集] --> B[标准化JSON]
    B --> C[运行质量门]
    C --> D[B1 / RiskDecision]
    D --> E[ChiefDecision]
    E --> F[正式报告模板]
    F --> G[结构校验与投递]
```

正式报告使用确定性脚本主链，不创建、不调用、不等待专业 Agent 或 Subagent。

盘前采用两段式窗口：08:50预采集公告/海外行情/RSS，09:05复用采集结果执行质量门、决策、渲染、校验和投递。

## 目录

- `00_governance`：总规则、工作流、角色边界、数据契约
- `01_data`：市场、板块、持仓、候选池等数据
- `02_agents`：逻辑职责、模板和数据结构；生产执行由确定性脚本承担
- `03_daily_plans`：面向用户的盘前日报和14:45报告；`_supporting`为忽略的运行中间产物
- `04_reviews`：正式日/周/月复盘；`*_review_draft.md`为忽略的临时草稿
- `05_strategy_versions`：策略版本记录
- `06_logs`：流水线运行日志
- `07_tools`：采集、分析、生成报告脚本

## 当前阶段

Phase 2：确定性生产主链。

## 关键规则

1. 个股服从板块，板块服从大盘。
2. 风控优先于买入。
3. stock_pool 负责选股，buy_strategy 负责买入计划。
4. risk_control 拥有否决权。
5. chief_decision 是最终交易计划输出层。
6. 所有计划必须可复盘。
