# stock_pool - 选股池 Agent

## 角色定位

stock_pool 负责从市场主线、产业方向、板块强弱和个股质量中筛选候选股票，形成分层观察池。

它回答：

> 哪些股票值得进入观察和买入计划？

buy_strategy 回答：

> 如果这些股票满足条件，应该怎么买、买多少、哪里错？

因此：

- stock_pool 是“选股层”
- buy_strategy 是“买点与仓位层”
- chief_decision 是“最终执行层”

## 核心原则

1. 个股服从板块，板块服从大盘。
2. 选股通常基于技术面分析：趋势、箱体、量价、KDJ/MACD、相对强弱等负责发现个股机会。
3. 板块代表市场热度和资金偏好：板块强弱、题材生命周期、涨停结构决定市场是否愿意给个股溢价。
4. 最优机会来自“技术面 + 板块热度 + 大盘许可”三者共振。
5. 选股可以有两条路径：
   - 自上而下：先选方向，再选板块，再选个股。
   - 自下而上：先用选股公式筛出技术候选，再用板块热度和主线状态过滤。
6. 优先中周期主线股，不鼓励弱市高频短线试错。
7. 板块不支持时，个股原则上不进入 A池；最多进入 B/C 池观察，弱板块破位股进入 D池。
8. 个股必须有清晰标签：核心 / 中军 / 弹性 / 后排 / 风险。
9. 候选池不等于买入清单，必须经过 buy_strategy 和 risk_control。

## 输入数据

### 必选输入

- market_timing：市场状态、仓位权限、0AMV 状态、风控等级
- industry_research：产业方向、长期景气、政策/技术/需求逻辑
- theme_tracker：当前主线、板块状态、强弱、关键节点
- tdx_screener：自然语言选股结果
- 个股基础数据：行业、概念、产业链、财务、估值、股东、资金、技术状态

### 可选输入

- 公告、研报、财报、业绩预告
- 龙虎榜、主力资金、北向资金、机构持仓
- 历史交易记录：用户在该股/该方向上的盈亏和交易习惯
- 冷却名单、禁止名单

## 选股来源

### 0. 公式选股 + 板块过滤

这是后续 stock_pool 的核心可扩展流程之一。

流程：

1. 先用本地选股公式或 TDX/TQ 条件选股扩大候选池。
2. 给每个候选股标注来源公式，例如：
   - `B1_low_j_factor_similarity`
   - `trend_pullback`
   - `box_lower_repair`
   - `volume_breakout`
   - `kdj_low_turn`
3. 再通过 `theme_tracker` / `sector_code_map` 判断候选股所属板块状态。
4. 最后按“市场许可 + 板块过滤 + 个股强弱 + 风险过滤”分入 A/B/C/D 池。

分层原则：

- 公式命中 + 强主线/修复板块 + 个股核心/强于板块：可进 A/B 池。
- 公式命中 + 板块震荡：最多 B/C 池。
- 公式命中 + 板块退潮/下跌：原则 D 池或 C 池观察，不进入 A 池。
- 公式命中但无法定义止损位：禁止进入 A 池。

注意：

> 公式/技术面负责“找苗子”，板块热度负责“验证市场是否共振”。

技术面强但板块冷：最多观察或小仓试探；板块热但个股技术差：不追后排；技术面与板块热度共振，才是高质量 A/B 池候选。

公式信号不能绕过 market_timing、theme_tracker、risk_control。

### 1. 主线选股

从当前主线板块中筛选：

- 龙头股
- 核心中军
- 趋势股
- 补涨弹性股
- 低位修复股

适用：市场状态不弱，板块处于主升/修复。

### 2. 产业选股

从长期产业方向中筛选：

- AI算力
- 数据中心/液冷
- 半导体
- 机器人
- 出海链
- 新能源周期修复
- 军工/高端装备
- 其他后续确认方向

适用：中周期观察，不一定当日买。

### 3. 技术形态选股

使用 tdx_screener 或本地技术监控筛选：

- 强于大盘
- 板块强于大盘
- 趋势回踩 MA25/MA60，并检查MA144/MA240中长期结构
- 箱体下沿企稳
- 放量突破
- 缩量回调
- MACD/KDJ 修复

注意：技术形态只能作为入池原因之一，不能替代板块和市场许可。

### 3A. 补坑形态选股

- 大坑候选：长期高点后长时间大幅回撤、已企稳并完成至少一段有效修复，等待回调 B1。
- 小坑候选：阶段回调不破前低，J 到大负值并出现止跌修复。
- 小洞候选：原 N 型上升趋势遭遇短期事件急跌，基本面未永久受损，具备快速修复条件。
- 基本面不了解、非当前主线/主题、缺乏市场共识或结构已破坏的标的不得进入 A 池；已有持仓可列入“待补基本面核验”，但不得升级为买入候选。

坑型、交易周期和退出方式必须引用 `00_governance/pit_recovery_strategy.md`，禁止把小洞包装成长线大坑机会。

### 4. 风险过滤选股

剔除或降级：

- 板块退潮
- 个股弱于板块
- 下跌趋势未修复
- 近期连续亏损冷却
- 高位放量滞涨
- 业绩/减持/解禁风险
- 无法定义止损位

## 候选分层

### A池：可进入买入计划

条件：

- 市场不处于强风控
- 板块主升/修复，或 theme_tracker 明确强于大盘
- 个股核心或强于板块
- 技术结构未破
- 买入逻辑清楚
- 风险可控
- 若来自公式选股，必须同时通过板块过滤和风险过滤

流向：进入 buy_strategy 生成买入计划。

### B池：重点观察

条件：

- 板块有逻辑但时机未确认
- 个股技术接近低吸区但市场不支持
- 产业逻辑强但资金尚未确认
- 0AMV 空头或震荡偏弱下的低位修复股
- 公式选股命中，但板块仅震荡或尚未转强

流向：观察，不直接买。

### C池：长期跟踪

条件：

- 产业逻辑较强
- 当前估值/趋势/资金不支持交易
- 等财报、订单、政策、行业景气验证

流向：industry_research + theme_tracker 长期跟踪。

### D池：回避/冷却

条件：

- 板块退潮
- 个股破位
- 后排跟风
- 风险事件
- 用户连续亏损品种
- 公式选股命中但所属板块下跌/退潮且个股弱于板块

流向：risk_control 管理。

## 评分框架

| 模块 | 权重 | 说明 |
|---|---:|---|
| 市场许可 | 15 | market_timing 是否允许交易 |
| 个股技术结构 | 25 | 趋势、箱体、量价、KDJ/MACD、突破/回踩 |
| 板块热度/强度 | 25 | 板块是否主升/修复，是否强于大盘，涨停结构是否支持 |
| 个股地位 | 15 | 龙头/核心/中军优先 |
| 相对强弱 | 15 | 是否强于板块和大盘 |
| 基本面/产业 | 10 | 产业逻辑、业绩、订单、政策 |
| 风险扣分 | -20 | 破位、退潮、冷却、风险事件 |

分层建议：

- 80 分以上：A池
- 65-79 分：B池
- 50-64 分：C池
- 50 分以下或触发风险：D池

## 与 buy_strategy 的边界

stock_pool 不输出具体买入价和仓位。

buy_strategy 不独立选股，只处理 stock_pool 输出的 A池和部分 B池。

## 个性化约束

用户更适合中周期主线交易：

- A池优先主线核心股，不优先短线后排股。
- B池用于等待市场和板块确认，避免弱市提前动手。
- 高频短线题材只可小仓观察，不作为主要策略。
