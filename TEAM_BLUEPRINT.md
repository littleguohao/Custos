# 投资策略 Team 蓝图

目标：建立一个持续进化的投资策略团队，用于辅助择时、选股、买入、卖出、持仓跟踪、复盘优化。

## 核心原则

1. 不追求单次判断正确，而追求系统长期正期望。
2. 所有建议必须可追踪、可复盘、可迭代。
3. 风控优先级高于收益预测。
4. 交易计划必须先定义无效条件，再定义盈利目标。
5. Team 输出的是决策辅助，不替代最终人工决策。

## Team 架构

### 1. 总控决策 Agent

职责：
- 汇总所有子 Agent 结论
- 统一仓位、风险预算和交易优先级
- 处理不同 Agent 之间的冲突
- 输出最终每日交易计划
- 记录决策依据，方便复盘

输出：
- 今日市场状态
- 总仓位建议
- 新开仓权限
- 持仓处理清单
- 风险提示

---

### 2. 市场择时 Agent

职责：
- 判断市场环境：进攻、震荡、防守、冰点
- 判断是否适合开新仓
- 给出总仓位区间

关注指标：
- 指数趋势
- 成交量
- 涨跌家数
- 涨停/跌停/炸板
- 连板高度
- 主线强度
- 北向/主力资金

输出：
- 市场状态
- 仓位建议
- 今日交易风格
- 禁止交易方向

---

### 3. 主线与题材 Agent

职责：
- 识别当前市场主线
- 判断题材生命周期
- 判断主线扩散、分歧、退潮

输出：
- 今日主线
- 强势板块
- 退潮板块
- 题材持续性判断

---

### 4. 选股池 Agent

职责：
- 从主线、自选股、条件公式、资金流向中筛选候选股
- 区分核心票、观察票、淘汰票

输出：
- 今日候选股池
- 入选理由
- 风险点
- 优先级排序

---

### 5. 买入策略 Agent

职责：
- 为候选股制定买入计划
- 明确触发条件、买入区间、首仓比例、加仓条件

输出：
- 买入触发价
- 买入确认信号
- 首仓比例
- 加仓条件
- 买入无效条件

---

### 6. 卖出风控 Agent

职责：
- 制定止损、止盈、移动止盈、逻辑证伪规则
- 识别连续亏损、执念交易、亏损拖延

硬规则：
- 同一股票连续亏损 2 次，冷却 10 个交易日
- 短线仓亏损 -5%~-7%，必须复盘或减仓
- 单票亏损超过 -10%，必须进入强制风控评估
- 当月短线胜率低于 35%，降低短线仓位

输出：
- 止损位
- 止盈位
- 减仓条件
- 清仓条件
- 是否触发冷却

---

### 7. 每日持仓研判 Agent

职责：
- 每日检查当前持仓
- 判断继续持有、加仓、减仓、止损、清仓

输入：
- 当前持仓
- 成本价
- 盈亏率
- 持仓天数
- 当日表现
- 板块强弱
- 个股公告/研报/异动

输出：
- 每只持仓动作建议
- 今日关键价位
- 明日验证点
- 风险等级

---

### 8. 交易复盘与进化 Agent

职责：
- 复盘历史交易
- 更新策略有效性
- 发现交易习惯问题
- 维护策略版本

关注：
- 胜率
- 盈亏比
- 最大回撤
- 持仓周期
- 买入后 MFE/MAE
- 卖出后走势
- 重复亏损品种

输出：
- 周复盘
- 月复盘
- 策略问题清单
- 策略修订建议

## 每日执行流程

当前标准流程：

1. 市场择时 Agent 判断环境、仓位上限、新开仓权限
2. 产业研究 Agent 更新长期产业方向和催化验证点
3. 主线与题材 Agent 判断主线、板块状态和退潮方向
4. 每日持仓研判 Agent 检查当前持仓
5. 选股池 Agent 生成 A/B/C/D 候选股分层
6. 买入策略 Agent 对 A池和部分 B池制定条件化买入/观察计划
7. 卖出风控 Agent 审核持仓风险和买入计划
8. 总控决策 Agent 输出最终每日交易计划
9. 收盘后复盘 Agent 记录执行结果并推动策略进化

详细工作流见：`00_governance/MASTER_WORKFLOW.md`

## 输出标准

每日必须输出：

- 今日市场状态
- 总仓位建议
- 今日可否开新仓
- 候选股票池
- 当前持仓处理建议
- 买入计划
- 卖出/风控计划
- 明日观察点

## 当前个性化约束

根据历史交易复盘：

1. 用户优势更偏中周期主线交易，而不是高频短线。
2. 2026 年 6 月亏损集中，短线试错密度过高。
3. 20 天以内交易贡献主要亏损。
4. 九丰能源等案例显示需要连续亏损冷却机制。
5. 卖出后 20 日平均收益为负，主要问题不是普遍卖飞，而是亏损单处理偏慢。
6. 风控 Agent 权重必须高于买入 Agent。

## 分阶段实施

### Phase 1：策略 Team 原型

- 固化 Team 架构
- 建立每日研判模板
- 接入持仓、自选股、交易记录
- 输出人工可读交易计划

### Phase 1.5：完整框架成型

- 接入 industry_research、stock_pool、buy_strategy、strategy_evolution 框架
- 明确 stock_pool 与 buy_strategy 的分工
- 建立总工作流、角色矩阵、每日运行手册、数据流契约和决策优先级
- 继续补充数据源、自动化稳定性和评分细节

### Phase 2：半自动化执行

- 每日自动生成市场状态
- 自动读取持仓与自选股
- 自动生成持仓处理建议
- 自动保存复盘日志

### Phase 3：策略进化

- 每周复盘策略表现
- 每月更新规则参数
- 维护策略版本号
- 淘汰失效规则，保留有效规则
