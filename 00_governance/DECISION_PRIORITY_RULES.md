# 决策优先级规则

日期：2026-07-09

## 总原则

当不同 Agent 结论冲突时，按以下优先级处理：

1. 风控纪律
2. 市场环境
3. 板块状态
4. 个股质量
5. 买点计划
6. 仓位效率
7. 短线机会

## 核心规则

### 1. 风控优先

只要 risk_control 触发以下任一情况，买入或加仓计划自动降级或取消：

- 个股在冷却名单
- 单票亏损超过阈值
- 技术破位未修复
- 板块退潮
- 无明确止损位
- 当月短线熔断

### 2. 市场许可优先

market_timing 为防守/冰点时：

- 原则不新开仓
- 不追高
- 不做突破买入
- 不因 J 值低位买入
- 只允许观察或极小仓位试探待总控确认

补充：

- 左侧趋势中，价格深跌、周线/月线 KDJ 的 J 值为负和低估值，只能提高观察优先级，不能越过市场许可和止跌确认直接买入。
- 右侧趋势中，日线 KDJ 的 J 值出现大负值，只能作为趋势内超跌候选，必须等待修复信号。
- 单独出现的 `J` 均指 KDJ 的 J 值。

0AMV 空头区间：

- 买入计划默认降级
- A池门槛提高
- 加仓需要强验证

### 3. 板块优先于个股

板块退潮：

- 后排股禁止买入
- 持仓进入风控观察
- 个股技术低位不构成买点

板块主升/修复：

- 只优先核心股、中军、强趋势股
- 后排补涨不作为主要策略

### 4. 选股优先于买点

没有进入 stock_pool A池的股票，不允许 buy_strategy 输出正常买入。

B池只能输出观察价位。

C池不生成交易计划。

D池禁止生成买入计划。

### 5. 止损先于买入

任何买入计划必须先定义：

- 技术止损位
- 亏损止损阈值
- 时间止损条件
- 逻辑无效条件

缺任何一项，不允许输出“允许买入”。

### 6. 加仓比买入更严格

加仓必须满足：

- market_timing 不低于震荡偏强
- 0AMV 不为空头
- 板块强于大盘
- 个股强于板块
- 首仓已有浮盈或逻辑被验证
- risk_control 未触发风控

禁止亏损摊低式加仓。

### 7. 持仓处理优先于新开仓

当存在破位/浮亏扩大持仓时：

- 先处理风险持仓
- 再考虑新开仓
- 总仓位不应因新买入而突破 market_timing 上限

## 典型冲突处理

| 场景 | 处理 |
|---|---|
| market_timing 防守，但个股 J < 12 | 观察，不买入 |
| 板块退潮，但个股接近箱体下沿 | 观察或回避，不低吸 |
| stock_pool A池，但无止损位 | 不允许买入 |
| buy_strategy 允许买入，但 risk_control 否决 | 取消买入 |
| 持仓浮亏 -8%，但产业逻辑好 | 先进入风控评估，不盲目补仓 |
| 个股突破，但板块分歧 | 不追高，等待确认 |
| 近期短线连续亏损，但出现短线热点 | 降低仓位或放弃 |

## 总控输出约束

chief_decision 最终输出必须包含：

- 允许动作
- 禁止动作
- 持仓处理优先级
- 买入计划是否执行
- 风控依据
- 明日验证点

如果存在冲突但无法判断，默认选择更保守方案。
