# 交易执行纪律

日期：2026-07-13

## 1. 波段卖出时间窗口

波段仓的计划性卖出原则上安排在交易日 `14:45` 之后执行，以收盘前确认信号为准，减少盘中洗盘、快速回撤和情绪恐慌造成的误卖。

适用范围：

- 趋势转弱但尚未触发硬止损
- 保护利润、主动减仓、普通调仓
- 波段策略的技术卖点确认

例外情况可以盘中立即处理：

- 触发预先定义的硬止损
- 买入逻辑被公告或事实证伪
- 重大风险事件、市场流动性快速恶化
- 跌停风险、连续快速下杀导致尾盘可能无法成交
- 用户预先定义的盘中强制退出条件

不得把“等待尾盘”作为拖延止损或放任风险扩大的理由。

### 每日持股检查纪律

已有持仓的普通卖出、减仓和清仓判断，必须先执行 `daily_holding_check_manual.md`：

- 忽略普通盘中冲高回落；冲刺阶段的一至两日小阴和快速洗盘，不得直接当作卖点。
- 下杀后先看尾盘是否快速收回，再看次日是否快速收回。
- 没有硬止损、重大风险或流动性恶化时，不在普通下杀中情绪化卖出。
- 大盘突然下杀时，若个股当天仍强势且只是同步下杀，不草率处理。
- 收盘为缩量小阴线且不破 BBI/结构低点时，再等一天确认。
- 收盘价抬高、不破 BBI、无巨量阴线、趋势向上、J 未有效死叉时，默认拿住。

硬止损、重大风险、流动性恶化、跌停风险、0AMV 空头反弹减仓和单票超限处理优先。

### 活跃实质空头区间的反弹处理

当 market_timing 基于有效 0AMV 数据确认市场处于活跃实质空头区间时，所有反弹首先视为减仓机会：

- 已有持仓应预先设置反弹减仓触发位、目标仓位和执行比例。
- 后排股、弱于板块个股、技术破位股和超仓持仓优先减仓。
- 不得借反弹加仓、摊低成本，也不得把单日上涨或盘中脉冲当作趋势反转。
- 普通计划性减仓原则上仍在 `14:45` 后确认；硬止损、重大风险或流动性恶化按盘中规则立即处理。
- 只有 0AMV 明确退出实质空头区间，并且大盘、板块和个股结构修复得到确认后，才恢复常规持有和买入评估。

## 2. 买入必须等待有效信号

任何买入和加仓都必须有可复核依据，不允许凭感觉、怕踏空、盘中脉冲或单一低位指标下单。

有效买入计划至少包含：

1. 所适配的走势状态：震荡向上 / 横盘震荡 / 震荡向下
2. 明确触发信号及确认口径
3. 买入价格区间
4. 首仓比例与风险预算
5. 无效条件和止损位
6. 信号未出现时的动作：继续等待，不提前埋伏

信号必须使用完成的 K 线或明确规定的实时确认时点。盘中短暂触及但收盘前未确认，默认不视为有效波段信号。

### 左侧与右侧信号差异

- 左侧（震荡向下）：先观察长期跌幅是否充分、周线/月线 KDJ 的 J 值是否为负或处于历史极低分位、估值是否足够低。上述条件只能进入重点观察，仍需止跌或结构修复后才能形成条件化试探计划。
- 右侧（震荡向上）：趋势结构未破时，观察日线 KDJ 的 J 值是否出现大负值或历史极低分位，并等待 J 拐头、支撑企稳、量价修复或 MACD 动能改善等信号。
- 横盘震荡：继续按照箱体位置和突破/跌破确认处理。
- 本体系后续单独出现的 `J` 均指 KDJ 的 J 值。

## 3. 浮盈转亏保护

持仓一旦形成有效浮盈，应记录该笔持仓的最大浮盈和保本退出线。股价从浮盈区回落至盈亏平衡保护带时，原则上执行卖出，不允许已盈利交易无计划地转为亏损交易。

### 盈亏平衡价

盈亏平衡价必须包含买入佣金、卖出佣金、印花税和预计滑点：

```text
保本价 = 持仓总成本（含买入费用） / 持仓数量
         + 预计卖出费用与滑点折算的每股金额
```

费用数据不完整时，暂以持仓系统的含费单位成本为基础，并增加保守费用缓冲；报告必须标记为估算值。

### 触发条件

- 先决条件：该笔持仓曾经进入有效浮盈，而非仅因盘口微小波动短暂翻红。
- 默认建议：最大浮盈达到 `+2%` 后启用保本保护。
- 保护带：最新可执行价格进入保本价上下 `0.3%` 区域，且尾盘仍未重新走强。
- 执行窗口：普通情况下在 `14:45` 后确认并卖出；若同时触发硬止损或重大风险，盘中立即处理。

`+2%` 和 `0.3%` 是首版参数，后续必须根据真实交易的 MFE/MAE、费用和滑点回测校准。

## 4. 每日记录要求

持仓跟踪至少记录：

- 含费成本价
- 当前盈亏率
- 持仓期间最大浮盈 MFE
- 是否已启用保本保护
- 保本价与保护带
- 当日 14:45 信号是否确认
- 最终动作和未执行原因
- 若处于活跃实质空头区间：反弹减仓触发位、计划比例、实际执行比例
