# 技术面选股 + 板块热度过滤工作流

日期：2026-07-09

## 核心思想

用户确认：选股通常是基于技术面的分析；板块是市场热度，能共振是最好的。

这条路径正式纳入 `stock_pool`：

> 公式/技术面负责“找苗子”，板块热度负责“验证市场是否共振”。

最优机会来自：

```text
个股技术面转强
+ 板块热度/主线支持
+ 大盘/0AMV许可
+ 风控可控
```

## 为什么这样做

单纯自上而下容易漏掉早期转强个股；单纯技术面/公式选股又容易在冷板块或退潮板块里选出假修复。

组合方式更稳：

1. 技术面扩大候选范围，发现个股机会。
2. 板块热度判断资金是否愿意给溢价。
3. 大盘/0AMV 判断环境是否允许交易。
4. 风控决定能否进入真实计划。

## 标准流程

### Step 1：技术面 / 公式选股

候选来源包括：

- `workflow_B1.py` / `run_b1_workflow.py`
- `tdx_screener`
- 后续自定义 TQ/TDX 公式
- 本地 `E:\O_DATA` 历史扫描

每个候选必须记录：

- `formula_id`
- `signal`
- `raw_rank`
- `formula_score`
- `source_file`

### Step 2：映射板块

对候选股补充：

- 行业
- 概念板块
- 产业链标签
- `theme_id`
- 对应 `sector_code_map` 中的主线方向

### Step 3：板块热度过滤

读取 `theme_tracker` 输出：

- `sector_state`
- `sector_score`
- `stage`
- `trade_permission`

过滤规则：

| 板块状态 | 公式候选处理 |
|---|---|
| 主升 / 修复，且强于大盘 | 可进入 A/B 池 |
| 震荡 | 最多 B/C 池 |
| 分歧 | 只看核心，最多 B 池 |
| 退潮 / 下跌 | 原则 D 池或 C 池观察 |
| 未知 | 不进 A 池 |

### Step 4：个股过滤

再判断：

- 是否核心 / 中军 / 弹性 / 后排
- 是否强于所属板块
- 是否破位
- 是否有止损位
- 是否存在风险事件
- 是否属于用户冷却名单

### Step 5：技术面 x 板块热度共振矩阵

| 个股技术面 | 板块热度 | 分层倾向 | 含义 |
|---|---|---|---|
| 强 | 强 | A/B池 | 最优共振，进入重点计划候选 |
| 强 | 中 | B池 | 个股先于板块，等待板块确认 |
| 强 | 弱 | B/C池 | 技术苗子，但市场不给溢价，谨慎观察 |
| 中 | 强 | B池 | 板块热，等个股技术确认，避免追后排 |
| 弱 | 强 | C/D池 | 板块热但个股弱，不追后排 |
| 弱 | 弱 | D池 | 双弱，回避 |

### Step 6：A/B/C/D 分层

| 分层 | 条件 |
|---|---|
| A池 | 技术面强 + 强板块/修复板块 + 个股核心/强于板块 + 风险可控 |
| B池 | 技术面强但板块待确认，或板块强但个股技术待确认 |
| C池 | 产业逻辑有，但技术面或板块热度不足 |
| D池 | 板块退潮、个股破位、风险事件、弱于板块 |

## 当前适用判断

2026-07-09 当前：

- `market_timing`：震荡偏弱，仓位 30%-50%。
- `theme_tracker`：半导体/芯片/存储/封测为唯一强主线。
- AI算力/服务器/液冷仍为退潮/下跌。

因此：

- 半导体强主线公式候选：可进入 A/B 池候选。
- AI算力公式候选：除非板块修复，否则最多 B/C 池。
- 证券、燃气、医疗设备等退潮板块公式候选：不进入 A 池。

## 禁止事项

- 禁止公式命中后直接买入。
- 禁止绕过 `theme_tracker` 板块过滤。
- 禁止绕过 `risk_control`。
- 禁止在没有止损位的情况下进入 A 池。

## 与 buy_strategy 的关系

`buy_strategy` 不处理原始公式候选。

只有进入：

- A池：生成完整买入计划。
- B池：生成观察价位和触发条件。

C/D 池不生成买入计划。
