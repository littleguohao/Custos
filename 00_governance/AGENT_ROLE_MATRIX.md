# Agent 角色矩阵

日期：2026-07-09

## 总览

| Agent | 层级 | 核心问题 | 主要输入 | 主要输出 | 是否有否决权 |
|---|---|---|---|---|---|
| market_timing | 市场许可 | 今天能不能做？做多少？ | 指数、0AMV、情绪、量能、外围 | 市场状态、仓位、新开仓权限 | 是 |
| industry_research | 产业方向 | 哪些产业长期值得看？ | 政策、产业、研报、财报、技术路线 | 长期方向、产业逻辑、验证点 | 否 |
| theme_tracker | 板块主线 | 哪些板块支持交易？ | 主线、板块走势、题材情绪 | 板块状态、强弱、关键节点 | 半否决 |
| stock_pool | 选股池 | 哪些股票值得进候选？ | 主线、产业、筛选器、个股资料 | A/B/C/D 候选池 | 半否决 |
| buy_strategy | 买入计划 | 候选股怎么买？ | stock_pool、技术位、市场许可 | 买入价、首仓、加仓、止损 | 否 |
| portfolio_review | 持仓研判 | 当前持仓怎么处理？ | 持仓、盈亏、技术、板块 | 持有/观察/减仓/止损建议 | 半否决 |
| risk_control | 风控纪律 | 哪些动作禁止？ | 全部计划、亏损、破位、冷却 | 禁止动作、风控清单 | 是 |
| chief_decision | 总控决策 | 今天最终做什么？ | 所有 Agent 输出 | 每日交易计划 | 是 |
| strategy_evolution | 复盘进化 | 规则是否有效？ | 计划、执行、结果 | 复盘、策略修订 | 否 |

## 权限说明

### 强否决权

以下 Agent 可以直接否决交易动作：

1. market_timing：市场防守/冰点、0AMV 空头时限制新开仓。
2. risk_control：触发止损、冷却、禁止加仓时否决交易。
3. chief_decision：根据总仓位、优先级和冲突处理做最终否决。

### 半否决权

以下 Agent 不直接出最终指令，但可以限制后续模块：

1. theme_tracker：板块退潮时限制相关个股。
2. stock_pool：D池股票禁止进入 buy_strategy。
3. portfolio_review：弱势持仓触发风控审核。

## 输入输出边界

### market_timing

不负责：选股、个股买卖点。

负责：市场状态、仓位上限、交易许可。

### industry_research

不负责：短线买点、日内技术交易。

负责：长期产业逻辑、方向优先级、产业验证点。

### theme_tracker

不负责：单只个股具体买入价。

负责：板块趋势、生命周期、关键支撑压力、是否支持个股交易。

### stock_pool

不负责：买入价、仓位、止损位。

负责：候选股分层和优先级。

### buy_strategy

不负责：独立选股、最终执行。

负责：对候选股制定条件化买入计划。

### portfolio_review

不负责：新开仓选股。

负责：当前持仓处理。

### risk_control

不负责：寻找机会。

负责：识别风险、限制错误动作。

### chief_decision

不负责：重复所有底层分析。

负责：冲突处理、最终计划、仓位分配。

## 冲突处理规则

| 冲突 | 处理 |
|---|---|
| buy_strategy 看多，但 risk_control 否决 | 以 risk_control 为准 |
| 个股技术好，但板块退潮 | 降级为观察或禁止 |
| 板块强，但 market_timing 防守 | 降低仓位或只观察 |
| stock_pool A池，但无明确止损 | 不允许买入 |
| 持仓浮亏扩大，但产业逻辑仍在 | 先风控，再复核逻辑 |
| 短线机会强，但用户近期短线亏损集中 | 降低仓位或放弃 |

## 用户个性化权重

根据交易复盘，当前权重排序：

1. 风控纪律
2. 市场环境
3. 板块主线
4. 个股质量
5. 买点性价比
6. 短线弹性

短线弹性不能压过风控纪律。
