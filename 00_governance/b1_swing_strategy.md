# B1 波段战法基础模型

日期：2026-07-13
版本：`B1-base-v1`

## 一、策略定位

B1 是应用于**上升趋势（右侧）**的波段策略。完整模型由六层组成：

1. 宏观与市场择时
2. 主线与个股筛选
3. KDJ 的 J 值触发
4. 结构止损
5. BBI 上方分批止盈
6. BBI/突破量价清仓

任何单一条件都不等于完整 B1 买点。尤其是 `J < 13`，只代表 B1 候选进入观察区，仍需通过择时、主题、个股和修复确认。

术语约定：本文及后续策略中单独出现的 `J`，均指 **KDJ 指标的 J 值**。

## 二、第一层：择时许可

B1 原则上只在股票处于震荡向上/右侧趋势时启用，并优先要求以下环境共振：

### 1. 经济周期

- 经济处于扩张或复苏向扩张阶段。
- “双宽”环境：财政政策偏积极、货币政策偏宽松。

### 2. 政策方向

- 方向符合国家中长期政策指引，例如十四五、十五五规划及其延续性产业政策。
- 政策必须与行业需求、资本开支或盈利改善具有可解释联系，不能仅凭概念标签。

### 3. 增量资金

- 0AMV 活跃市值处于多头区间，代表市场存在增量资金许可。
- 0AMV 为空头区间时，B1 默认降级为观察，不执行正常仓位买入。

### 4. 流动性环境

- 降息、降准或其他流动性宽松措施有利于风险偏好与估值扩张。
- 流动性宽松是加分项，不替代个股和板块信号。

### 择时结论

输出三档：

- `允许`：右侧趋势 + 宏观/政策支持 + 0AMV 多头，允许进入后续选股和买点判断。
- `受限`：部分条件支持但增量资金或市场状态未确认，只观察或极小仓试探。
- `禁止`：市场强风控、0AMV 空头、板块退潮或个股非右侧趋势，不执行 B1。

## 三、第二层：选股许可

B1 候选优先满足：

1. 属于当前市场主线或有持续性的市场主题。
2. 有明确政策支持，且政策能够映射到行业或公司基本面。
3. 个股是行业龙头、主线核心、中军或高辨识度标的。
4. 存在资金流入或异动证据，例如成交额放大、相对板块走强、主力资金改善或关键价位资金承接。
5. 个股三状态识别为震荡向上，或回调后仍保持右侧结构。

主题与主线采用不同周期：

- 普通主题经常只有一波行情，优先等待**日线 B1**，交易周期更短，仓位和止盈更保守。
- 市场主线持续性更强，优先等待**周线 B1**，再用日线寻找执行点。

## 四、第三层：B1 买点

### 1. 基础触发

```text
KDJ 的 J < 13
```

周期口径：

- 主题交易：日线 `J < 13`，称为日线 B1 候选。
- 主线交易：周线 `J < 13`，称为周线 B1 候选；实际下单仍需日线确认。

### 2. 买入确认

`J < 13` 后必须继续等待至少一类修复信号：

- J 低位拐头向上
- 当前 K 线或随后 K 线收回关键支撑
- 缩量止跌或放量修复
- MACD 绿柱缩短、低位金叉或底背离
- 重新站回短期均线或回踩关键位不破
- 个股相对板块重新转强

只有以下条件同时成立，才形成可执行 B1 买入计划：

```text
择时允许
AND 主线/主题与政策许可
AND 龙头/核心与资金许可
AND 对应周期 J < 13
AND 修复信号确认
AND 止损位可定义
```

信号未确认时继续等待，不因担心踏空提前买入。

## 五、第四层：止损

买入前必须确定结构止损，候选位置二选一或取更合理者：

1. 当前买入信号 K 线的最低点。
2. N 型结构中上一个有效低点。

确定止损时还要检查：

- 当前价格所在的上下区间、箱体和支撑阻力。
- 收盘价是否有效跌破，而非只看盘中瞬时影线。
- 止损距离是否与仓位风险预算匹配。

波段技术止损原则上在 14:45 后按收盘前状态确认；重大风险、快速下杀、跌停风险或预设硬止损可以盘中立即执行。

## 六、第五层：止盈与减仓

当股价运行在 BBI 线上方，并连续出现两根中阳线或大阳线时，可以开始分批减仓、锁定利润。

标准 BBI 暂按通达信常用口径：

```text
BBI = (MA3 + MA6 + MA12 + MA24) / 4
```

执行原则：

- “两根中大阳线”是启动减仓条件，不要求一次清仓。
- 减仓比例结合涨幅、成交量、距离前高、J 过热程度和板块强弱决定。
- 已形成有效浮盈后，同时启用盈亏平衡保护，防止赢转亏。

### 中大阳线量化口径

中大阳线必须首先是阳线，即 `收盘价 > 开盘价`。再根据股票对应的日涨跌幅限制，使用约半个涨停幅度作为门槛：

| 日涨跌幅限制 | 中大阳线门槛 |
|---:|---:|
| 10% | 单日涨幅或阳线实体幅度 >= 5% |
| 20% | 单日涨幅或阳线实体幅度 >= 10% |
| 30% | 单日涨幅或阳线实体幅度 >= 15% |

计算口径：

```text
单日涨幅 = 收盘价 / 前收盘价 - 1
阳线实体幅度 = (收盘价 - 开盘价) / 开盘价
```

当单日涨幅或阳线实体幅度达到对应门槛时，记为一根中大阳线；连续两个交易日均满足且股价位于 BBI 上方，触发分批减仓。股票涨跌幅限制必须按证券当日实际交易规则识别，不能仅凭代码静态猜测。

## 七、第六层：清仓

满足任一条件，进入清仓评估：

### 1. BBI 破位

- 股价连续两个交易日收盘跌破 BBI 线。
- 原则上第二个交易日 14:45 后确认并执行；硬风险例外。

### 2. 突破前高后的量价异常

- 以突破前高的交易日为 `T`。
- `T+1` 为突破后的第一个交易日。
- 在 `T+1` 收盘后判断，不在盘中提前确认。
- 若 `T+1` 为阴线，即 `收盘价 < 开盘价`，则该日成交量定义为“阴量”；阳线，即 `收盘价 > 开盘价`，其成交量定义为“阳量”。
- 若 `T+1` 为阴线，且 `T+1成交量 / T成交量` 接近 `75%`，触发清仓信号。

“接近 75%”的中心值已确定为 `0.75`，但允许偏差尚未定型。系统在未完成样本回测前必须同时输出实际量比；若需要自动化布尔触发，应将容差作为可配置参数，不得静默假设。

停牌、无有效成交或数据缺失不计为交易日，也不得生成量价清仓信号。

## 八、状态机

```text
不允许
  -> 择时允许
  -> B1候选（对应周期 J < 13）
  -> 修复确认
  -> 已买入
  -> BBI上方两根中大阳线：分批止盈
  -> 连续两日收盘跌破BBI / 突破后量价异常：清仓
```

任一阶段出现市场强风控、板块退潮、逻辑证伪或硬止损，直接转入风险退出，不必等待正常状态迁移。

## 九、每日必备字段

| 层级 | 字段 |
|---|---|
| 择时 | 经济周期、财政/货币状态、政策方向、0AMV状态、市场许可 |
| 选股 | 主线/主题、政策证据、个股地位、资金异动、三状态走势 |
| 买点 | 日线J、周线J、J是否<13、J是否拐头、修复信号 |
| 风控 | 信号K线最低点、N型前低、计划止损价、风险预算 |
| 止盈 | BBI、价格与BBI关系、当日涨跌幅限制、单日涨幅、阳线实体幅度、连续中大阳线数量、最大浮盈 |
| 清仓 | 连续跌破BBI天数、前高突破日T、T+1日期、T+1阴阳线、T+1/T量比 |

## 十、边界

1. B1 是右侧波段模型，不用于未修复的左侧下跌趋势。
2. `J < 13` 是候选触发，不是无条件买入指令。
3. 普通主题与市场主线必须使用不同周期和持仓预期。
4. 所有买入必须在下单前定义止损和仓位。
5. 所有卖出遵循 14:45 波段确认纪律，但硬风险优先。
6. 参数必须通过多股票、跨行业、跨市场阶段回测后再定型。
