# 盘后复盘六步工作流

日期：2026-07-13

## 核心顺序

每日盘后复盘必须按以下顺序进行：

```text
新闻 -> 大盘 -> 热点 -> 持仓 -> 选股 -> 次日交易计划
```

前一层是后一层的过滤条件。不得先决定买卖，再倒推新闻、热点或指标作为理由。

## 1. 看新闻

覆盖四类信息：

- 信息：公司、行业、海外市场和产业事件
- 政策：宏观、财政、货币、监管及产业政策
- 风向：资金偏好、风险偏好和市场叙事变化
- 舆情：市场关注度、分歧、过热和风险传播

当日高优先级宏观发布必须单独检查，不得只依赖RSS候选新闻。至少覆盖：

- 海关总署：进出口、出口、进口、贸易顺差、重点出口产品结构。
- 国家统计局：CPI、PPI、PMI、工业增加值、社零、固定资产投资、GDP。
- 中国人民银行/金融监管：社融、信贷、M1/M2、公开市场、利率和流动性。
- 财政部/发改委/商务部：财政收支、专项债、产业政策、消费和外贸政策。

若当日存在上述高优先级宏观发布，盘后复盘必须写入“事实-影响-交易含义”；无法取得时必须在数据缺失项中明示“宏观发布未核验”。

输出要求：

- 区分事实、观点和未确认消息
- 记录时间、来源及影响对象
- 判断利多/利空、影响周期和是否已被市场定价
- 映射至大盘、热点、持仓和候选股

## 2. 看大盘

回答市场是否允许交易以及允许多少仓位：

- 主要指数和风格指数涨跌、趋势与关键位置
- 市场宽度、成交额、涨跌停和情绪
- 0AMV 活跃市值的当日值与跨日有效状态
- 增量资金、流动性和风险偏好
- 市场三状态及总仓位上限

市场许可优先于热点和个股信号。

## 3. 看热点

区分：

- 市场主线：持续性强、容量大、政策/产业逻辑明确
- 市场主题：持续周期可能较短，常见一波行情
- 新热点：首次形成资金共振，尚需确认持续性
- 过时热点：催化衰减、资金退潮、核心股破位或市场已充分交易

每个热点至少回答：

1. 当前属于主线、主题、候选新热点还是过时方向。
2. 是否有政策、产业、业绩或事件支撑。
3. 是否持续，资金是否仍在流入。
4. 龙头、核心、中军和后排分别是谁。
5. 次日需要验证什么，什么情况判定退潮或失效。

## 4. 看持仓

逐只检查：

- 原始持有逻辑是否仍成立
- 所属大盘、板块和个股三状态
- 当日盘面关键点和相对板块强弱
- 是否出现买入、卖出、止损、止盈信号
- 是否出现加仓或减仓时机
- 最大浮盈、含费保本价和赢转亏风险
- 当前仓位和交易后预计仓位是否超过20%

持仓处理优先于新开仓。计划性波段卖出原则上在14:45后确认，硬止损和重大风险例外。

## 5. 选股

同时保留两条路径：

- 自动选股：本地通达信/TQ、条件选股、公式、B1及其他模型
- 手动选股：根据新闻、政策、主线、主题、行业研究和盘面观察加入

所有候选统一去重并经过：

1. 大盘许可
2. 热点/板块过滤
3. 个股地位和资金过滤
4. 三状态走势匹配
5. 风险事件和冷却名单过滤
6. A/B/C/D分层

自动或手动来源只表示发现路径，不能绕过相同的买入和风控规则。

## 6. 制定次日交易计划

次日计划必须回答：

- 总仓位目标与上限
- 每只持仓：持有、买入、加仓、减仓、止损或清仓
- 买什么、卖什么
- 买多少、卖多少
- 触发价格或触发信号
- 无效条件和止损位
- 执行时间，尤其14:45卖出确认
- 开盘、盘中、尾盘三种情景及应对动作

精确数量必须建立在持仓、价格和账户总权益均为新鲜且已确认的数据上；数据不完整时只给比例、方向和触发条件。

## 报告质量三级门

- `blocked`：交易记录/无交易确认或当日0AMV任一核心事实缺失，不生成正式复盘，只列阻断项。
- `degraded`：核心事实齐全，但技术指标、情绪、成交额、板块、账户权益等辅助数据部分缺失。仍须生成完整六步复盘，逐项标注缺失，并禁止由缺失数据支持精确数量或提高交易权限。
- `complete`：核心事实与所需行情、账户和分析数据均为当日已确认口径，可生成完整复盘及精确数量计划。

技术行情过期、市场情绪/成交额缺失、账户总权益仅可估算，均属于 `degraded`，不得单独将整份复盘升级为 `blocked`。

## 单票20%硬上限

任何一只股票的交易后仓位不得超过账户总权益的20%：

```text
单票仓位 = 该股票交易后市值 / 账户交易后总权益
单票仓位 <= 20%
```

执行规则：

1. 20%是最高上限，不是建议目标。
2. 若市场、策略或风险规则给出更低上限，执行更低值。
3. 新买和加仓前必须计算交易后预计仓位；超过20%的计划自动否决或缩减数量。
4. 当前单票已经超过20%时，禁止继续买入或加仓，并进入减仓评估。
5. 因价格上涨被动超过20%时，不机械盘中砍仓，但必须在最近一次14:45复核中制定恢复至20%以内的计划；硬风险优先处理。
6. 持仓或账户权益数据过期时，不得声称精确满足20%上限，也不得输出精确买入数量。

## 盘后复盘输出结构

1. 新闻、政策、风向和舆情
2. 大盘、指数、资金与市场许可
3. 主线、主题、新热点和过时热点
4. 持仓逐只复盘与仓位审计
5. 自动/手动选股及A/B/C/D分层
6. 次日仓位与条件化交易计划
7. 今日纪律偏差、有效规则和待验证参数
8. 数据时效、缺失项和风险提示
