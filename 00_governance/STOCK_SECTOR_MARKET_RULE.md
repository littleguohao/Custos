# 策略规则：个股-板块-大盘层级约束

日期：2026-07-09

## 核心规则

个股服从板块，板块服从大盘。

任何个股买入、持有、加仓、减仓、止损判断，都不能只看个股本身，必须按以下层级确认：

1. 大盘环境是否支持交易？
2. 所属板块是否处于强势、修复、震荡、退潮？
3. 个股是否强于板块？
4. 个股是否是板块核心票、跟风票，还是后排补涨？
5. 板块形态是否支持继续持有？

## 层级优先级

### 1. 大盘优先

若大盘处于防守或冰点：

- 降低所有个股仓位上限
- 限制新开仓
- 个股利好不能对抗系统性风险
- 持仓重点转为风控，而不是进攻

### 2. 板块其次

若板块处于退潮：

- 即使个股短期抗跌，也不能盲目加仓
- 后排股优先减仓
- 核心股只允许观察或被动持有，必须设置更紧风控

若板块处于主升或强修复：

- 个股机会有效性提高
- 核心票可提高优先级
- 后排票仍需控制仓位

### 3. 个股最后

个股判断必须回答：

- 是否强于所属板块？
- 是否有独立逻辑？
- 是否是板块核心？
- 如果板块转弱，个股是否还能成立？

## 持仓板块跟踪要求

每日持仓研判必须跟踪每只持仓的所属板块：

| 项目 | 必须判断 |
|---|---|
| 所属主板块 | 个股当前主要跟随哪个板块 |
| 板块趋势 | 上升 / 震荡 / 破位 / 退潮 |
| 板块位置 | 主升 / 分歧 / 修复 / 高位滞涨 / 退潮 |
| 板块强度 | 强于大盘 / 跟随大盘 / 弱于大盘 |
| 个股地位 | 龙头 / 核心 / 中军 / 跟风 / 后排 |
| 个股相对强弱 | 强于板块 / 同步板块 / 弱于板块 |

## 操作规则

### 加仓条件

只有同时满足：

- 大盘不处于防守/冰点
- 板块强于大盘
- 个股强于板块
- 个股仍处于核心地位

才允许加仓。

### 持有条件

至少满足：

- 大盘没有系统性破位
- 板块没有明显退潮
- 个股没有明显弱于板块
- 买入逻辑未证伪

### 减仓条件

出现任一情况，应考虑减仓：

- 大盘转弱，市场进入防守
- 板块跌破关键趋势
- 板块内核心股补跌
- 个股弱于板块
- 后排股在板块分歧中放量滞涨

### 清仓/止损条件

出现任一情况，进入强风控：

- 大盘防守/冰点 + 板块退潮
- 板块主线证伪
- 个股跌破关键位且弱于板块
- 个股从核心退化为跟风/后排

## 对 Agent 的影响

### market_timing

负责判断大盘环境，决定市场是否支持交易。

### theme_tracker

必须跟踪板块趋势、题材生命周期、板块强弱排序。

### portfolio_review

必须逐只持仓分析所属板块走势和个股相对板块强弱。

### risk_control

当“大盘弱 + 板块弱 + 个股弱”三重共振时，必须提高风控等级。

### chief_decision

最终交易计划必须遵守：

个股机会不得凌驾于板块和大盘之上。

## 2026-07-09 补充：板块与个股分析粒度

用户补充：个股还需要更多更细的因子，板块主要判断走势以及关键节点。

已新增规则：

`00_governance/SECTOR_STOCK_ANALYSIS_GRANULARITY.md`

原则：

- 板块是过滤器，主要判断走势、阶段、关键支撑/压力、突破/破位节点

- 板块支持不代表所有个股都能买
- 板块不支持时，个股除非有强独立逻辑，否则不加仓
