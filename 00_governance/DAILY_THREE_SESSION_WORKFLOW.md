# 交易日三时点工作流

## 一、持仓数据如何维护

### 唯一交易事实源

- `01_data/trades/master_trade_ledger.csv`：只增不改的全量交易主台账
- `01_data/trades/daily_new_trades_template.csv`：每日只补新增成交
- `01_data/trades/current_positions.json`：由增量成交维护数量与成本

Excel 全量文件只用于历史迁移、对账和灾备重建，不再作为每日必需输入。详细规则见 `INCREMENTAL_TRADE_LEDGER.md`。

### 收盘重估

持仓数量与成本来自交易台账；收盘价、市值、盈亏和仓位必须用当日收盘行情重估。

### 关键限制

当前不是券商账户实时同步。B1 策略默认盘中不交易；任何盘中交易只有在用户主动告知，或 Excel/增量台账更新并重新导入后才进入持仓事实源。因此：

1. 所有报告必须展示持仓导入时间。
2. 14:45 默认沿用上一确认收盘持仓作为盘中持仓基线；若检测到目标日成交或用户说明盘中交易，则必须先更新持仓再生成建议。
3. 14:45 的默认无交易基线可以用于持仓处理、减仓和精确数量建议；不得据此放宽加仓或新开仓权限。
4. 不允许用行情涨跌自行推测买卖数量或持仓变化。
5. 用户盘后补充交易记录后，必须先重新导入，再做复盘。

## 二、盘前预警（交易日 08:30）

### 输入

- 隔夜外围指数及科技链表现
- 重要公告、业绩预告、监管/政策信息
- 突发事件及可能的 A 股传导路径
- 最近交易日 market_timing、theme_tracker
- 当前持仓及观察池

### 输出

`03_daily_plans/YYYY-MM-DD_premarket_alert.md`

固定回答：

- 隔夜风险偏好及 A 股可能映射
- 持仓相关的重要公告/突发事件
- 可能强化或削弱的主线
- 盘前风险清单
- 开盘后 30 分钟验证点

无法确认的消息不得写成事实；历史数据必须标注日期。

## 三、14:45 操作建议（交易日 14:45）

### 输入

- `current_positions.json` 及 `_import_meta.json`
- 持仓个股实时行情
- 大盘指数、持仓所属板块和主线实时走势
- 当日市场宽度、情绪和成交
- 最新风险证据与当日交易计划

### 输出

`03_daily_plans/YYYY-MM-DD_1445_review.md`

固定回答：

- 当前持仓快照时间与是否可能过期
- 持仓实时盈亏与强弱
- P1/P2/P3 处理优先级
- 收盘前应持有、减仓、止损或保护利润的标的
- 是否允许新开仓
- 尾盘和次日风险

执行纪律：

- 波段仓计划性卖出原则上在 14:45 后确认，硬止损和重大风险除外。
- 14:45 必须使用尾盘实时价，按 B1 默认盘中不交易的持仓基线输出当前持仓操作建议。
- 逐只检查是否由有效浮盈回落至含费保本保护带，防止赢转亏。
- 买入和加仓必须严格等待预设信号；未触发时保持观察。
- 具体口径遵循 `TRADING_EXECUTION_DISCIPLINE.md`。

不能使用硬编码股票或价格；触发位来自当日技术与风险数据。

## 四、盘后复盘（用户补数后触发）

### 用户补充

1. 当天新增交易记录，或明确确认“今日无交易”
2. 当日 0AMV（活跃市值）收盘值
3. 如有必要，说明计划外交易原因

### 执行顺序

1. 将新增成交追加至主交易台账，并核对持仓数量与成本。
2. 写入当日 0AMV。
3. 采集收盘指数、市场宽度、涨跌停、板块和持仓表现。
4. 重跑 market_timing、theme_tracker、portfolio_review、risk_control、chief_decision。
5. 对照盘前和 14:45 建议，复盘是否命中、是否偏离。
6. 输出 `04_reviews/daily/YYYY-MM-DD_final_review.md/json`。
7. 更新下一交易日观察点。

### 复盘至少包含

- 新闻：信息、政策、风向、舆情及其影响映射
- 大盘：市场、指数、资金、0AMV和交易许可
- 热点：主线、主题、持续性、新热点和过时热点
- 持仓：持有逻辑、盘面关键点、买卖点及加减仓时机
- 选股：自动/手动候选、过滤和A/B/C/D分层
- 次日计划：总仓位、买卖对象、比例/数量、触发条件和无效条件
- 当日交易及持仓变化、计划执行偏差和盈亏归因
- 有效规则、失效规则和数据问题

单只股票交易后仓位不得超过账户总权益的20%；数据不新鲜时不得输出精确数量。详细流程见 `AFTER_CLOSE_REVIEW_WORKFLOW.md`。

## 五、调度边界

- 盘前和 14:45 可以自动运行并主动推送。
- 盘后复盘不能在用户未补充交易记录和 0AMV 时自动假定数据；应先提醒补数，收到后立即执行。
- 法定节假日即使是周一至周五，也应检查是否交易日；非交易日跳过并保持安静。
- 交易日历由 `07_tools/trading_calendar.py` 通过本地通达信 HTTP `get_trading_dates` 每周刷新，默认覆盖当月初至未来370天；人工 `overrides` 优先级最高。
- 通达信服务暂时不可用时保留最近一次成功缓存，不得清空日历；缓存覆盖范围外的工作日保持 `unknown` 并 fail-closed。
# 盘前日报固定信息结构

交易日盘前日报必须优先回答以下三类问题，且使用上一个交易日收盘后至当日开盘前的绝对时间窗口：

1. 市场重大消息与持仓相关消息/公告：逐条标注时间、来源、证据质量及利好/利空/中性。权威交易所公告可标记为 confirmed；普通媒体和 RSS 仅作 candidate，不得单独提高交易权限。
2. 美国、日本、韩国重要指数：至少包括道琼斯工业指数、标普500、纳斯达克综合指数、日经225、韩国KOSPI，必须给出点位、具体涨跌百分比、数据时间，并区分“已收盘”与“盘中最新”。
3. 持仓状态与计划调整：逐只对照最近一次盘后复盘，展示 MA25/MA60/MA144/MA240、日线J值、原交易计划、隔夜新增证据和“维持/收紧/不放宽”的明确结论。

隔夜利空可以收紧原计划；隔夜利好只能增加观察或等待价格确认，不得自动把禁止加仓改为允许加仓。若消息或行情缺失，日报必须明确写“未检索到/待确认”，但仍应生成并投递。
