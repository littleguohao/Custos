# 每日运行手册

日期：2026-07-09

## 目标

把策略 Team 每天稳定跑起来，形成可执行、可复盘的交易计划。

## 盘前流程

### Step 1：市场择时

产出：`market_timing` 报告。

运行顺序：

1. `market_timing_collector.py` 采集指数、快照、外围等基础数据。
2. 人工或后续自动补入 0AMV 收盘涨跌幅。
3. `apply_screener_overlay.py` 用 `tdx_screener` 收盘口径补齐涨停、跌停、炸板、连板高度。
4. `market_timing_scorer.py` 生成最终评分报告。

必须回答：

- 市场状态：进攻 / 震荡偏强 / 震荡偏弱 / 防守 / 冰点
- 0AMV 状态：做多 / 中性 / 空头
- 涨停、跌停、炸板率、连板高度
- 建议总仓位
- 是否允许新开仓
- 风控等级

未完成则：后续买入计划默认只观察。

### Step 2：产业与主线

产出：`industry_research` + `theme_tracker`。

运行顺序：

1. `theme_tracker_report.py` 读取 `01_data/sectors/sector_code_map.json`。
2. 批量计算重点板块趋势、箱体、日/周/月 KDJ、MACD。
3. 输出 `01_data/sectors/YYYY-MM-DD_sector_technical_summary.json`。
4. 输出 `03_daily_plans/YYYY-MM-DD_theme_tracker.md`。
5. 总控必须读取 theme_tracker 结论，避免仅凭个股技术信号加仓。

必须回答：

- 今日主线是什么？
- 哪些板块支持交易？
- 哪些板块退潮/风险？
- 持仓所属板块是否支持继续持有？

### Step 3：持仓研判

产出：`portfolio_review`。

必须回答：

- 每只持仓动作：持有 / 观察 / 减仓 / 止损 / 清仓
- 是否弱于板块
- 是否破位
- 是否触发亏损阈值
- 标准 BBI（MA3/6/12/24均值）、收盘在 BBI 上/下、连续收盘跌破天数及持股提醒：BBI上方维持技术持有；首日跌破验证次日收回；连续两日跌破进入清仓评估。0AMV、硬止损、重大风险和单票超限等更高优先级规则仍有效。

### Step 4：选股池

产出：`stock_pool`。

运行思路：

1. 自上而下：从 `theme_tracker` 强主线/强板块中找核心股。
2. 自下而上：用本地选股公式或 TDX/TQ 条件选股先筛技术面候选。
3. 对技术候选执行板块热度过滤：
   - 技术面强 + 板块热度强：允许进入 A/B 池。
   - 技术面强 + 板块热度中：进入 B 池观察。
   - 技术面强 + 板块热度弱：最多 B/C 池，等待板块确认。
   - 技术面弱 + 板块热度强：最多 B/C 池，避免追后排。
   - 板块退潮/下跌：原则 D 池或 C 池观察。
4. 再结合个股地位、相对强弱、风险项分层。

必须输出：

- A池：可进入买入计划
- B池：重点观察
- C池：长期跟踪
- D池：回避/冷却

若 market_timing 防守或 0AMV 空头：A池必须非常严格，大多数候选进 B/C 池。公式信号不能绕过板块过滤。

### Step 5：买入策略

产出：`buy_strategy`。

只处理：

- A池：完整买入计划
- B池：观察价位和触发条件

必须包含：

- 买入条件
- 买入价区间
- 首仓比例
- 加仓条件
- 无效条件
- 止损位
- 时间止损

缺止损位，不允许输出可买。

### Step 6：风控审核

产出：`risk_control`。

必须回答：

- 哪些持仓要风控？
- 哪些买入计划要否决？
- 哪些股票/方向进入冷却？
- 今日禁止动作是什么？

### Step 7：总控决策

产出：`chief_decision`。

最终输出：

- 今日总仓位
- 今日允许动作
- 今日禁止动作
- 持仓处理优先级
- 可执行买入计划
- 仅观察方向
- 明日验证点

## 盘中执行规则

1. 只执行 chief_decision 计划内动作。
2. 临时机会必须满足：市场许可 + 板块支持 + 个股入池 + 风控未否决。
3. 禁止无计划追高。
4. 禁止因 J 值低位直接补仓。
5. 禁止亏损摊低式加仓。
6. 若触发止损位，优先执行风控。
7. 对已有持仓，先按每日持股检查手册过滤盘中冲高回落、同步下杀、缩量小阴线和快速洗盘；没有硬风险时，不在普通下杀中草率卖出。

## 收盘后流程

### Step 1：记录执行

记录：

- 实际买入/卖出
- 是否按计划执行
- 偏离计划原因
- 情绪/冲动交易标记

### Step 2：复盘计划质量

检查：

- market_timing 是否有效
- stock_pool 入池是否合理
- buy_strategy 买点是否清晰
- risk_control 是否及时
- chief_decision 是否过于激进或保守

### Step 3：更新明日观察

输出：

- 明日关键指数位
- 明日关键板块
- 明日重点持仓
- 明日候选股变化
- 明日风险点

## 每日最小可运行版本

如果时间不足，至少完成：

1. market_timing
2. portfolio_review
3. risk_control
4. chief_decision

选股和买入策略可以只输出观察，不强行交易。

## 成功标准

连续 10 个交易日后检查：

- 是否每天都有计划？
- 是否减少计划外交易？
- 是否减少亏损拖延？
- 是否减少弱市高频试错？
- 是否形成可迭代数据？
