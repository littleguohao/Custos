# 补坑策略：大坑、小坑与小洞

日期：2026-07-14
版本：`pit-recovery-v1`

## 一、策略定位

补坑策略先观察股票长期价格形态，再决定交易周期和收益预期。形态分为：

1. 大坑：长期大幅回撤后的中长期修复。
2. 小坑：局部 B1 调整后的短中期波段。
3. 小洞：上升趋势中黑天鹅急跌后的单段反弹。

三类形态不能混用。小洞不能按大坑长期持有；大坑也不能因为一次 J 大负值就直接抄底。

## 二、大坑：长期修复型

### 形态定义

- 股票曾创历史新高或重要长期高点。
- 随后经历较长时间连续下跌，股价大幅回撤。
- 下跌速度明显放缓，并已企稳一段时间，不再持续创新低。
- 基本面未被永久性破坏，行业或公司仍有长期修复逻辑。

### 策略目标

- 目标是长期持有并赚取大波段修复收益。
- 初次企稳只进入观察，不直接重仓。
- 优先等待股票先完成一段有力度的填坑行情，证明资金与市场认知开始修复；随后回调形成 B1，再考虑条件化买入。

### 失效条件

- 企稳区或长期关键低点被有效跌破。
- 盈利能力、资产质量、行业空间或公司竞争力发生永久性恶化。
- 第一段修复无量、无法突破关键压力，随后重新进入长期下跌。

## 三、小坑：局部 B1 波段型

### 形态定义

- 股票先形成阶段高点，之后连续回调并逐步企稳。
- 回调不有效跌破本轮上升结构的阶段前低。
- 日线 J 到大负值或个股自身历史极低分位。
- J 大负值后出现拐头、缩量止跌、MACD 动能改善或重新站回关键位等修复确认。

### 策略目标

- 目标是赚取局部小波段，不预设为长期反转。
- 止损优先设在阶段前低、N 型前低或信号 K 线最低点。
- 到达前高、箱体压力或出现 BBI/动能退出信号时分批兑现。

## 四、小洞：事件急跌反弹型

### 形态定义

- 股票原本处于连续 N 型上升趋势，低点和高点逐级抬高。
- 因可验证的黑天鹅或突发事件驱动突然大跌。
- 日线 J 到大负值或历史极低分位。
- 事件属于短期冲击，未永久破坏公司基本面、行业逻辑和原上升结构。

### 策略目标

- 只做快进快出的一段反弹，主要目标是抢反弹的一根有效 K 线。
- 必须等待止跌或反包确认，禁止在急跌途中徒手接刀。
- 首根有效反弹、事件预期兑现或未按预期快速修复时退出，不转成长线持仓。

## 五、什么样的坑能买

以下条件作为完整许可链，缺少关键项时只能观察：

1. 大坑企稳后先观察；优先等待完成第一段有力度的填坑，再在回调形成 B1 时考虑。
2. 新买入必须理解基本面。基本面不了解、无法解释下跌原因或无法判断是否永久受损的股票不买。
3. 行业题材必须属于当前市场共识的炒作主题或主线；非当前主线、无持续性或纯后排题材不买。
4. 大坑形成后，至少已经完成一个中等规模子坑的修复，证明结构具备实际填坑能力。
5. 填坑过程中 N 型结构清晰，高点和低点逐步抬高，回调不破关键前低。
6. 标的具有可验证的市场共识，包括主线地位、行业地位、基本面逻辑、资金承接或持续催化中的至少两类证据。

## 六、基本面不了解的持仓处理

用户原始表述为“基本面不了解的不卖”。本体系拆分为两种场景：

- 新买入：基本面不了解的不买，不能用技术形态替代基本面判断。
- 已有持仓：基本面不了解时不因恐慌或单日波动直接卖出；必须先补充基本面核验，判断下跌是否永久破坏逻辑。
- 若已有持仓同时触发硬止损、重大风险、板块退潮、0AMV 空头反弹减仓或单票超限等风控规则，仍按 risk_control 和 chief_decision 执行，不能用“不了解不卖”绕过风控。

## 七、统一否决条件

- J 大负值只能说明超跌，不能单独构成买点。
- 大跌幅不能证明已经见底；企稳时间、结构修复和基本面必须同时验证。
- 0AMV 处于活跃实质空头区间时，本策略默认只观察；已有持仓的反弹优先用于减仓，不得借“填坑”名义扩大仓位。
- 板块退潮、个股弱于板块、风险事件永久改变基本面、无法定义止损位时禁止执行。
- 单票仓位、总仓位和短线熔断继续服从 risk_control 与 chief_decision，不因形态好看而放宽。

## 八、标准输出字段

| 字段 | 说明 |
|---|---|
| pit_type | 大坑 / 小坑 / 小洞 / 不符合 |
| prior_high | 历史或阶段前高 |
| drawdown_pct | 相对前高最大回撤 |
| stabilization_period | 企稳持续时间 |
| prior_structural_low | 阶段前低或 N 型前低 |
| j_state | J 值及历史分位 |
| first_recovery_completed | 是否已完成第一段填坑 |
| n_structure | N 型结构是否完整 |
| fundamental_status | 已理解 / 待核实 / 已破坏 |
| holding_fundamental_action | 新买入否决 / 持仓补核验 / 风控处理 |
| theme_status | 主线 / 主题 / 非主线 / 退潮 |
| consensus_evidence | 市场共识证据 |
| trade_horizon | 长期大波段 / 局部小波段 / 单段快速反弹 |
| entry_trigger | 条件化入场信号 |
| invalidation | 结构与逻辑失效条件 |

## 九、策略优先级

执行优先级始终为：

`风险控制 > 0AMV与大盘许可 > 主线/板块 > 基本面 > 坑型识别 > B1/J值等执行信号`

坑型识别负责定义机会类型和持有周期，不负责绕过市场许可直接下单。
